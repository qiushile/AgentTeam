## 二、工单处理流程

### 2.1 标准流程
```
用户提交工单
    ↓
AI自动分类 (技术问题/计费问题/安全事件/一般咨询)
    ↓
    ├→ 技术问题 → Dev团队处理 → 解决 → 用户确认 → 关闭
    ├→ 计费问题 → 运营团队核对 → 回复 → 用户确认 → 关闭
    ├→ 安全事件 → 安全团队处理 → 回复 → 用户确认 → 关闭
    └→ 一般咨询 → 客服自动回复/人工处理 → 关闭
```

### 2.2 工单模板

**技术问题工单**:
```

内容:

```

**计费问题工单**:
```

内容:
```

## 三、常见问题处理话术

### 3.1 创建失败
**用户**: "实例创建一直失败"
**标准回复**:
> 您好，已收到您的反馈。请提供实例ID，我将为您查询具体原因。常见原因包括：余额不足、GPU库存紧张、系统维护中。如为余额问题，充值后即可正常创建。

### 3.2 计费疑问
**用户**: "为什么费用比预期高？"
**标准回复**:
> 您好，我来帮您核对账单。请提供实例ID和使用时间段，我将为您查询详细计费记录。我们的计费精确到秒，您可以查看每笔扣费明细。

### 3.3 GPU性能
**用户**: "GPU性能不如预期"
**标准回复**:
> 您好，GPU性能受多种因素影响。请执行以下检查：
> 1. 运行 `nvidia-smi` 确认GPU型号和驱动正常
> 2. 检查是否有其他进程占用GPU
> 3. 确认使用的深度学习框架已正确安装CUDA支持
> 如仍有问题，请提供 `nvidia-smi` 截图，我将为您进一步排查。

## 五、升级机制

### 5.1 升级路径
```
一线客服 → 二线技术 → 三线开发 → 技术负责人
```

### 5.2 升级条件
| 升级 | 条件 |
|------|------|
| 一线→二线 | 技术问题超过30分钟未解决 |
| 二线→三线 | 需要代码级修复的问题 |
| 三线→负责人 | 影响超过10个用户的重大问题 |
