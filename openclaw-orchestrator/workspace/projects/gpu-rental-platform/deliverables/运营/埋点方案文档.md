## 一、埋点架构

### 1.1 数据流转
```
前端/后端 → 埋点SDK → Kafka → 
  ├→ ClickHouse（OLAP实时分析）
  ├→ Hadoop（离线数仓）
  └→ Redis（实时指标缓存）
```

### 1.2 埋点方式
| 方式 | 用途 | 实现 |
|------|------|------|
| 代码埋点 | 核心业务事件 | 前端SDK + 后端中间件 |
| 全埋点 | 页面访问、点击 | 前端SDK自动采集 |
| 可视化埋点 | 运营活动页面 | 管理后台配置 |

## 三、后端埋点

### 3.1 API调用事件
```go
// 中间件自动采集
type APICallEvent struct {
    Endpoint     string        `json:"endpoint"`
    Method       string        `json:"method"`
    ResponseTime time.Duration `json:"response_time"`
    StatusCode   int           `json:"status_code"`
    UserID       string        `json:"user_id"`
    IP           string        `json:"ip"`
```

## 四、数据模型

### 4.1 事件表结构 (ClickHouse)
```sql
CREATE TABLE events (
    event_time DateTime64(3),
    event_date Date,
    user_id String,
    session_id String,
    event_name LowCardinality(String),
    event_params Map(String, String),
    page_url String,
    referrer String,
    device_type LowCardinality(String),
    os LowCardinality(String),
    browser LowCardinality(String),
    ip String
) ENGINE = MergeTree()
PARTITION BY toYYYYMM(event_date)
ORDER BY (event_date, event_name, user_id, event_time);
```

### 4.2 用户表结构
```sql
CREATE TABLE users (
    user_id String,
    register_date Date,
    register_channel LowCardinality(String),
    user_type LowCardinality(String),
    last_login_date Date,
    total_recharge Decimal(10,2),
    total_spend Decimal(10,2),
    instance_count UInt32,
    updated_at DateTime
) ENGINE = ReplacingMergeTree()
ORDER BY user_id;
```

## 五、埋点SDK集成

### 5.1 前端SDK
```javascript
// 初始化
import { GPUCloudTracker } from '@gpucloud/tracker';

const tracker = new GPUCloudTracker({
  appId: 'gpu-cloud-web',
  serverUrl: 'https://tracker.gpucloud.com/collect',
  autoTrack: true, // 自动采集page_view
});

// 手动埋点
tracker.track('instance_create_start', {
  gpu_model_id: 5,
  image: 'ubuntu-22.04-pytorch',
});

// 用户标识
tracker.identify('user-123', {
  user_type: 'enterprise',
  register_channel: 'organic',
});
```

### 5.2 后端中间件 (Go)
```go
// Gin中间件自动采集API调用
func TrackingMiddleware(tracker *Tracker) gin.HandlerFunc {
    return func(c *gin.Context) {
        start := time.Now()
        c.Next()
        
        event := APICallEvent{
            Endpoint:     c.FullPath(),
            Method:       c.Request.Method,
            ResponseTime: time.Since(start),
            StatusCode:   c.Writer.Status(),
            UserID:       c.GetString("user_id"),
            IP:           c.ClientIP(),
        }
        tracker.Track("api_call", event)
    }
}
```

## 六、数据质量保障

### 6.1 埋点验证
| 验证项 | 方法 | 频率 |
|--------|------|------|
| 事件完整性 | 对比需求文档与实际埋点 | 每次发版 |
| 数据准确性 | 抽样核对业务数据 | 每日 |
| 参数完整性 | 检查必填参数是否缺失 | 实时告警 |
| 时序正确性 | 检查事件时间是否合理 | 实时 |

### 6.2 异常处理
- 埋点SDK失败不影响主业务流程
- 本地缓存+异步上报
- 上报失败重试3次
- 丢弃率监控告警
