# 运营Agent（数据分析部）任务清单

## 任务 1：数据指标体系搭建
**优先级**: P0 | **截止时间**: 第6周

### 核心指标定义

#### 1.1 用户增长指标
| 指标 | 计算方式 | 目标值 |
|------|----------|--------|
| 新增注册用户数 | 当日新注册用户数 | 月均+15% |
| 获客成本(CAC) | 营销费用 / 新增付费用户 | ≤500元 |
| 激活率 | 注册后24h内创建实例的用户占比 | ≥30% |
| 付费转化率 | 付费用户 / 总注册用户 | ≥15% |

#### 1.2 业务核心指标
| 指标 | 计算方式 | 说明 |
|------|----------|------|
| 活跃实例数 | 运行中GPU实例总数 | 实时 |
| GPU利用率 | 已分配GPU算力 / 总GPU算力 | 目标≥75% |
| 平均在线时长 | 用户实例日均运行小时数 | 衡量粘性 |
| 续费率 | 包月用户续费比例 | 目标≥70% |
| ARPU | 月营收 / 月活跃付费用户 | 追踪趋势 |

#### 1.3 收入指标
| 指标 | 计算方式 | 说明 |
|------|----------|------|
| MRR | 月度经常性收入 | 订阅类收入 |
| 月营收 | 当月所有订单金额 | 含按量付费 |
| 毛利率 | (营收 - GPU成本) / 营收 | 目标≥40% |
| LTV | 用户平均生命周期 × ARPU | 用户价值 |
| LTV/CAC | 用户价值 / 获客成本 | 目标≥3 |

## 任务 2：埋点方案设计
**优先级**: P0 | **截止时间**: 第5周

### 事件埋点清单
| 事件名 | 触发时机 | 关键参数 |
|--------|----------|----------|
| page_view | 页面加载 | page_url, referrer, user_id |
| user_register | 注册成功 | channel, user_type |
| user_login | 登录成功 | user_id, device |
| instance_create | 提交创建实例 | gpu_model, image, price |
| instance_start | 启动实例 | instance_id, gpu_model |
| instance_stop | 停止实例 | instance_id, runtime_hours |
| instance_release | 释放实例 | instance_id, total_cost |
| recharge | 充值成功 | amount, payment_method |
| jupyter_open | 打开Jupyter | instance_id |
| ssh_connect | SSH连接 | instance_id |
| billing_view | 查看账单 | period |
| api_call | API调用 | endpoint, response_time, status |

### 埋点SDK集成
- 前端：自研埋点SDK或采用 GrowingIO / Sensors Data
- 后端：Go中间件自动记录业务事件
- 数据流转：Kafka → Flink(实时) / ClickHouse(离线分析)

## 任务 3：数据仓库架构
**优先级**: P0 | **截止时间**: 第7周

### 架构设计
```
业务数据库(PostgreSQL) → CDC → Kafka → 
  ├→ ClickHouse（OLAP实时分析）
  ├→ Hadoop/Data Lake（离线数仓）
  └→ Redis（实时指标缓存）
```

### 数据分层
- **ODS层**: 原始数据接入
- **DWD层**: 数据清洗、维度建模
- **DWS层**: 轻度聚合（日/周/月粒度）
- **ADS层**: 应用层（直接供BI看板使用）

## 任务 4：BI数据看板
**优先级**: P1 | **截止时间**: 第8周

### 看板清单（Grafana/Metabase）

#### 4.1 运营总览大屏
- 实时活跃实例数（大数字）
- 今日营收（金额 + 环比）
- GPU利用率仪表盘
- 新增用户趋势（7日折线图）
- 热门GPU型号排行

#### 4.2 用户分析看板
- 用户增长漏斗（注册→激活→付费→续费）
- 用户留存曲线（7日/30日/90日）
- 用户分群画像（按行业/规模/地区）
- 流失用户预警

#### 4.3 收入分析看板
- 日/周/月营收趋势
- 按GPU型号收入分布
- 按时段收入热力图
- 欠费/坏账统计

#### 4.4 资源利用率看板
- GPU节点在线状态
- 各节点负载分布
- 闲置资源识别
- 容量规划预测

### 报表自动化
- 日报：每日9:00自动推送至飞书
- 周报：每周一推送周度分析
- 月报：每月1号推送月度经营分析

## 任务 5：异常预警机制
**优先级**: P1 | **截止时间**: 第8周

### 预警规则
| 预警项 | 触发条件 | 通知方式 |
|--------|----------|----------|
| GPU利用率过低 | 连续2h < 30% | 飞书 + 邮件 |
| 收入异常下降 | 日营收环比下降 > 30% | 飞书 + 短信 |
| 用户流失异常 | 日活跃下降 > 20% | 飞书 |
| 欠费率上升 | 欠费用户占比 > 10% | 飞书 + 邮件 |
| 实例异常率 | 创建失败率 > 5% | 飞书 + 电话 |
