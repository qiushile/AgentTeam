# Prompt 工程最佳实践

## 1. 基本原则

### 1.1 明确具体
- ✅ "写一个 PostgreSQL 连接池配置，最大连接数 200，事务模式"
- ❌ "写个数据库配置"

### 1.2 提供上下文
- ✅ "基于 Node.js 20 + Express 4，实现一个带 JWT 认证的用户登录 API"
- ❌ "写个登录 API"

### 1.3 指定输出格式
- ✅ "输出 JSON 格式的 ESLint 配置，包含以下规则..."
- ❌ "写个配置"

## 2. Prompt 模板

### 2.1 代码生成
```
角色: 你是 [角色] 专家
任务: [具体任务描述]
技术栈: [技术/框架/版本]
输入: [输入数据/接口定义]
输出要求: [输出格式/约束条件]
示例: [可选的输入输出示例]
```

### 2.2 代码审查
```
审查代码:
```[语言]
[代码内容]
```

审查维度:
1. 安全性 (SQL注入/XSS/CSRF)
2. 性能 (N+1查询/内存泄漏)
3. 可维护性 (命名/结构/注释)
4. 测试覆盖

输出: 问题列表 + 修复建议
```

### 2.3 Bug 修复
```
问题描述: [错误信息/现象]
复现步骤:
1. ...
2. ...

相关代码:
```[语言]
[代码]
```

期望行为: [描述]
实际行为: [描述]
```

## 3. 高级技巧

### 3.1 Few-Shot Prompting
提供 2-3 个示例，让 AI 学习模式:
```
示例 1:
输入: "CREATE TABLE users ..."
输出: "CREATE INDEX idx_users_email ON users(email);"

示例 2:
输入: "SELECT * FROM tasks WHERE ..."
输出: "CREATE INDEX idx_tasks_status ON tasks(status);"

任务:
输入: "SELECT * FROM orders WHERE user_id = ..."
输出:
```

### 3.2 Chain of Thought
要求 AI 逐步推理:
```
请按以下步骤分析:
1. 识别问题
2. 分析根因
3. 列出备选方案
4. 推荐最优方案
5. 给出实现代码
```

### 3.3 约束条件
明确禁止的行为:
```
要求:
- 使用 TypeScript
- 不使用 eval()
- 必须包含错误处理
- 添加 JSDoc 注释

禁止:
- 不使用 any 类型
- 不使用 console.log (使用 logger)
- 不硬编码配置
```

## 5. 质量评估

### 5.1 评估标准
| 维度 | 评分 | 说明 |
|------|------|------|
| 准确性 | 1-5 | 输出是否正确 |
| 完整性 | 1-5 | 是否覆盖所有需求 |
| 可执行性 | 1-5 | 代码是否可直接运行 |
| 安全性 | 1-5 | 是否有安全隐患 |

### 5.2 反馈循环
- 好的 prompt → 保存到 Prompt 库
- 差的输出 → 分析原因，优化 prompt
- 持续迭代
