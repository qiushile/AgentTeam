# 研发安全基线检查清单

## 1. 代码安全规范

### 1.1 OWASP Top 10 防护
| 风险 | 防护措施 | 检查方式 |
|------|---------|---------|
| SQL 注入 | 参数化查询 / ORM | ESLint `sql-no-unsafe-queries` |
| XSS | 输出编码 / CSP | ESLint `react/no-danger` |
| CSRF | SameSite Cookie / Token | 代码审查 |
| 敏感数据泄露 | 环境变量 / 密钥管理 | 静态扫描 |
| 不安全的反序列化 | 白名单校验 | 代码审查 |
| 组件漏洞 | 依赖更新 / 扫描 | Dependabot |
| 认证失效 | JWT / Session 管理 | 渗透测试 |
| 授权缺失 | RBAC / ABAC | 代码审查 |

### 1.2 密钥管理
- [ ] **禁止**在代码中硬编码密码、API Key、Token
- [ ] 使用环境变量或密钥管理服务 (HashiCorp Vault / AWS Secrets Manager)
- [ ] `.env` 文件必须加入 `.gitignore`
- [ ] 定期轮换密钥（建议 90 天）

### 1.3 输入校验
- [ ] 所有外部输入必须校验 (类型、长度、格式)
- [ ] 使用 Joi / Zod / Yup 等校验库
- [ ] 文件上传限制大小、类型

## 2. 依赖漏洞扫描

### 2.1 扫描工具链
| 工具 | 用途 | 频率 |
|------|------|------|
| `npm audit` | Node.js 依赖漏洞 | 每次 install |
| `yarn audit` | Yarn 依赖漏洞 | 每次 install |
| Snyk | 多语言依赖扫描 | 每日 CI |
| Dependabot | 自动 PR 修复 | 实时 |
| Trivy | 容器镜像漏洞 | 每次构建 |

### 2.2 自动化扫描 (GitHub Actions)
```yaml
- name: Dependency Audit
  run: npm audit --audit-level=high
  
- name: Snyk Scan
  uses: snyk/actions/node@master
  env:
    SNYK_TOKEN: ${{ secrets.SNYK_TOKEN }}
```

### 2.3 漏洞分级响应
| 级别 | CVSS 评分 | 响应时间 |
|------|----------|---------|
| Critical | 9.0 - 10.0 | 24 小时 |
| High | 7.0 - 8.9 | 72 小时 |
| Medium | 4.0 - 6.9 | 2 周 |
| Low | 0.1 - 3.9 | 下个迭代 |

## 3. 渗透测试流程

### 3.1 测试范围
- [ ] Web 应用 (前端 + 后端 API)
- [ ] 数据库权限提升测试
- [ ] Docker 容器逃逸测试
- [ ] 网络端口暴露检查

### 3.2 测试工具
| 工具 | 用途 |
|------|------|
| OWASP ZAP | Web 应用扫描 |
| Burp Suite | 手动渗透测试 |
| Nmap | 端口扫描 |
| sqlmap | SQL 注入测试 |
| Nikto | Web 服务器扫描 |

### 3.3 测试频率
- **内部**: 每次 major release 前
- **外部**: 每年 1-2 次专业机构

## 4. 基础设施安全

### 4.1 网络隔离
- [ ] 数据库仅允许应用服务器 IP 访问
- [ ] 禁止公网直接暴露数据库端口
- [ ] 使用 VPC / 安全组隔离

### 4.2 容器安全
- [ ] 非 root 用户运行容器
- [ ] 只读文件系统 (readOnlyRootFilesystem)
- [ ] 最小化镜像 (Alpine/Distroless)
- [ ] 定期更新基础镜像

### 4.3 访问控制
- [ ] 数据库最小权限原则
- [ ] API 认证 + 授权
- [ ] SSH 密钥登录 (禁用密码)

## 5. 数据安全

### 5.1 加密
| 数据类型 | 加密方式 | 说明 |
|---------|---------|------|
| 传输中数据 | TLS 1.2+ | HTTPS/WSS |
| 静态数据 | AES-256 | 数据库加密字段 |
| 密码 | bcrypt / Argon2 | 不可逆哈希 |
| API Key | 环境变量 + KMS | 运行时解密 |

### 5.2 数据脱敏
- [ ] 日志中不记录敏感信息（密码、Token、身份证号）
- [ ] 测试环境使用脱敏数据
- [ ] 错误信息不暴露内部实现细节

## 6. 安全事件响应

### 6.1 响应流程
```
发现 → 确认 → 隔离 → 修复 → 验证 → 复盘
```

### 6.2 角色分工
| 角色 | 职责 |
|------|------|
| 发现者 | 报告安全事件，保留证据 |
| 安全工程师 | 确认影响范围，制定修复方案 |
| 研发负责人 | 协调修复资源，批准上线 |
| 创始人 | 重大事件决策，外部沟通 |
