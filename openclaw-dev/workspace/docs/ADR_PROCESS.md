# ADR 决策流程

## 1. 何时需要 ADR

当以下情况发生时，应创建 ADR:
- 技术选型 (框架、数据库、中间件)
- 架构变更 (微服务拆分、API 设计)
- 重要配置变更 (部署策略、安全策略)
- 团队开发规范变更

## 2. ADR 生命周期

```
提议 (Proposed) → 讨论 (Discussing) → 接受 (Accepted) → 执行中 (Implementing) → 已完成 (Completed)
                                                                    ↓
                                                            已弃用 (Deprecated)
```

## 3. 创建 ADR 流程

1. **复制模板**: 复制 `docs/ADR_TEMPLATE.md` 为 `docs/adr/ADR-NNNN-标题.md`
2. **填写内容**: 按模板填写上下文、备选方案、决策、后果
3. **提交 PR**: 提交到仓库，请求团队 review
4. **标记状态**: review 通过后更新状态为 "Accepted"
5. **执行**: 按决策实施方案
6. **更新索引**: 更新 `docs/ADR_TEMPLATE.md` 底部的 ADR 索引

## 5. 决策原则

- **简单优先**: 选择最简单的方案，除非有明确理由
- **可逆性**: 优先选择容易撤销的决策
- **数据驱动**: 用数据支持决策，而非主观判断
- **文档化**: 所有决策必须记录，包括被否决的方案
